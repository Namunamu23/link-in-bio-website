I want you to help me design, build, test, and publish a custom link-in-bio website for my girlfriend, who is a digital artist and 2D animator.

I will attach reference images showing the design and visual style she wants. Treat those images as the primary design reference. Carefully inspect all visible details, including:

* Layout and spacing
* Typography
* Colors and gradients
* Illustrations and decorative elements
* Buttons, links, cards, and icons
* Alignment and proportions
* Desktop and mobile behavior
* Animations and interactive details
* The overall artistic mood and personality

## Main objective

Create a polished, responsive, production-ready personal website that presents her artwork, animation work, social media accounts, portfolio links, contact information, and any other links we decide to include.

The result should feel like a professionally designed custom website—not a generic Linktree template or an obviously AI-generated project.

Do not merely imitate the reference images mechanically. Understand the design system behind them and translate it into a functional website while preserving the intended style.

## How I want you to work

Treat this as a complete web-development project rather than giving me a quick code sample.

Work through the project in clear stages:

1. Analyze the reference images and explain what you observe.
2. Identify any missing information required to build the site correctly.
3. Research and recommend the most suitable technology stack.
4. Create the project structure and implementation plan.
5. Build the website with complete, usable code.
6. Help me run and test it locally.
7. Review and improve the design with me.
8. Optimize accessibility, responsiveness, performance, and SEO.
9. Help me deploy it.
10. Help me purchase and connect a custom domain.

Do not jump directly into writing code before analyzing the design and confirming the important content and functionality.

Ask only questions that materially affect the design or implementation. Group the questions together instead of repeatedly interrupting the workflow with one question at a time.

When a minor detail is missing, make a reasonable temporary assumption, clearly label it, and continue. Do not invent personal information, URLs, artwork descriptions, usernames, or contact details.

## Research requirements

Before recommending the implementation, research the current professional approaches used for small portfolio and link-in-bio websites.

Prioritize:

* Official documentation
* Current browser standards
* Official framework and library documentation
* Reliable accessibility and web-performance guidance
* Current hosting and domain-provider information
* Well-maintained open-source resources when relevant

Use current information rather than relying on outdated tutorials.

You do not need to research every technology that exists. Research only the options that are realistically relevant to this project, compare them briefly, and select the simplest professional solution that satisfies the requirements.

For factual claims involving current pricing, product limitations, hosting policies, domain renewals, framework versions, or deployment procedures, verify the information online and cite the source.

Clearly distinguish between:

* Verified facts
* Your professional recommendation
* Temporary assumptions
* Decisions that require my approval

## Technology selection

Recommend a technology stack based on the actual needs of the project, not merely because a framework is popular.

The site will probably be mostly static, so avoid unnecessary backend systems, databases, authentication, or complex infrastructure unless the requested features genuinely require them.

Consider options such as:

* Semantic HTML, modern CSS, and JavaScript
* React, Next.js, Astro, or another suitable framework
* TypeScript
* Modern CSS approaches
* Lightweight animation libraries
* Static-site generation
* Image optimization
* Free or inexpensive hosting platforms

Explain the important tradeoffs in plain English before choosing.

Favor:

* A simple and maintainable structure
* Fast loading
* Minimal dependencies
* Easy deployment
* Easy future editing
* Strong mobile support
* Good accessibility
* Low or zero recurring hosting costs

Use stable, actively maintained technologies and current official documentation. Do not add libraries when native HTML, CSS, or JavaScript can handle the requirement cleanly.

## Design and originality requirements

The finished site should feel personal to a digital artist and animator.

Avoid common signs of a generic AI-generated website, including:

* Excessive gradients without a design purpose
* Generic glassmorphism
* Random glowing elements
* Overused stock layouts
* Corporate startup-style language
* Unnecessary feature sections
* Placeholder marketing copy
* Excessive rounded cards
* Random icons
* Inconsistent spacing
* Animations that distract from the artwork
* Code comments that sound like an AI tutorial
* Generic variable names and unnecessary abstractions

Instead, create a coherent visual system based on the reference design:

* Consistent spacing
* Intentional typography
* Thoughtful responsive behavior
* Custom visual details
* Natural microinteractions
* Clear hierarchy
* Strong presentation of her artwork
* Subtle animations that fit a 2D animator’s portfolio
* A layout that does not feel copied from a standard template

Do not claim that the website was handmade by a human or attempt to misrepresent how it was produced. The goal is simply to create thoughtful, customized, professional-quality work.

## Animation requirements

Because she is a 2D animator, movement can be part of the website’s identity, but it must remain tasteful and efficient.

Animations should:

* Support the design rather than cover weak layout decisions
* Remain smooth on ordinary mobile devices
* Avoid causing layout shifts
* Respect the user’s reduced-motion preference
* Avoid blocking access to content
* Avoid autoplaying loud audio or intrusive video
* Use lightweight techniques whenever possible

Explain whether CSS animations, the Web Animations API, GSAP, Motion, Lottie, or another option is appropriate before adding a dependency.

## Quality standards

The finished website should:

* Work on current mobile and desktop browsers
* Look intentional at narrow, medium, and wide screen sizes
* Load quickly on mobile connections
* Avoid horizontal scrolling
* Preserve artwork quality without loading unnecessarily large files
* Have no obvious console errors
* Have no broken links
* Be navigable without a mouse
* Remain understandable when animations are disabled
* Use valid metadata and page titles
* Avoid cumulative layout shift where practical
* Be easy for me to maintain after deployment

Use tools such as Lighthouse or equivalent browser-development tools to evaluate the completed site. Do not chase a perfect score by damaging the design, but identify and fix meaningful problems.
